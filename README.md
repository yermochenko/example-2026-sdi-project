# Пример проекта по дисциплине "Введение в разработку ПО"

## Вступительная кампания

Приложение должно позволять пользователю:

1. Формировать список (просматривать, добавлять, изменять и удалять элементы) специальностей университета с указанием:

- шифра специальности;
- названия специальности;
- краткого названия специальности (аббревиатуры);
- квалификации, присваиваемой выпускникам этой специальности;
- планируемого количества мест набора;
- списка предметов, по которым нужно сдать ЦТ для поступления на эту специальность, а также нижний порог оценки по каждому предмету ЦТ, при котором абитуриент допускается до участия в конкурсе на поступление.

2. Формировать список абитуриентов с указанием:

- ФИО;
- оценок по каждому сданному ЦТ (целое число от 0 до 100);
- среднего балла аттестата (дробное число от 0 до 10 с одним знаком после запятой).

Программа после добавления нового абитуриента должна назначать ему уникальный целочисленный идентификатор. Идентификаторы назначаются последовательно начиная с 1.

После добавления нового абитуриента и при просмотре информации о существующем абитуриенте приложение должно показывать список доступных специальностей, на которые абитуриент может поступать (может быть допущен до участия в конкурсе), который формируется на основе совпадения сданных абитуриентом ЦТ со списком предметов по специальности и их нижним порогам ЦТ.

3. Назначать абитуриента (или отменять назначение) кандидатом на зачисление на одну (и только одну) из доступных ему специальностей.

4. Инициировать (запустить) процедуру проведения конкурса на зачисление кандидатов (абитуриентов, назначенных на специальность) на выбранную специальность. При этом программа должна вывести список зачисленных кандидатов и:

- если количество кандидатов больше, чем плановое количество мест, то вывести проходной/полупроходной балл и конкурс (количество человек на одно место);

- если количество кандидатов меньше или равно плановому количеству мест, то вывести количество не занятых мест (недобор на специальность).

При проведении конкурса на зачисление кандидатов для каждого кандидата вычисляется его суммарный балл: сумма всех его оценок по ЦТ и среднего балла аттестата, умноженного на 10. Кандидаты зачисляются на специальность в порядке убывания их суммарного балла. Балл последнего зачисленного кандидата считается проходным. Если же несколько кандидатов с одинаковым суммарным баллом претендуют на оставшееся количество мест, которое меньше числа претендующих кандидатов, то этот суммарный балл называется полупроходным. В случае получения полупроходного балла, кандидаты с таким баллом зачисляются в порядке возрастания их уникальных идентификаторов.
